# Contributing to ViNNi

Thank you for your interest in contributing to ViNNi (Virtually Integrated Neural Network Intelligence). Before submitting any contributions, please read and follow these guidelines carefully.

---

## 1. Attribution & Authorship

ViNNi is created and maintained by **Abhishek Arora**.  

By contributing, you acknowledge:
- Abhishek Arora is the original creator of ViNNi.
- All contributions will preserve authorship and attribution.
- The project license (Non-Commercial) applies to all contributions.

---

## 2. Usage Restrictions

ViNNi is free for:
- Personal use  
- Educational use  
- Research and non-commercial experimentation  

**Commercial or business use requires explicit written permission from the creator.**  
This includes:
- Paid products or services  
- SaaS or API integrations  
- Internal company use  
- Training/fine-tuning for commercial deployment  

---

## 3. Pull Request Workflow

All changes must go through **pull requests (PRs)**. Direct pushes to the `main` branch are not allowed.  

**PR Requirements:**
- Include a clear **title** summarizing the change.  
- Provide a **description** with:
  - Purpose of the change  
  - Related issue or task (if applicable)  
  - Test coverage notes  
- Assign a label: `bug`, `feature`, `docs`, `refactor`, `experimental`.  

**Reviews:**
- At least **one review approval** is required.  
- Stale approvals will be dismissed if new commits are added.  
- Sensitive or security-related changes may require two reviewers.  

**Merge:**
- Only **squash merging** is allowed to keep the history clean.

---

## 4. Code Quality

All contributions must adhere to:
- Formatting standards and linting  
- Proper testing where applicable  
- No intentionally malicious or unsafe code  

---

## 5. Security Reporting

If you discover a security vulnerability or unsafe behavior:
- Do **not** open a public issue.  
- Report it **privately** to the creator.  
- Include:
  - Description of the issue  
  - Steps to reproduce  
  - Potential impact  
  - Suggested mitigation (optional)  

Responsible disclosure ensures ViNNi remains safe for all users.

---

## 6. Thank You

Your contributions help make ViNNi better while respecting the project’s rules and creator’s rights.
